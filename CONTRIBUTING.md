# Contributing

Thanks for your interest in contributing!

## Getting started

1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/my-feature`).
3. Install dependencies: `pip install -r requirements.txt`
4. Make your changes.
5. Run lint: `ruff check backend/`
6. Run tests: `pytest`
7. Commit and push, then open a Pull Request.

## Guidelines

- Follow existing code style (the linter will catch most issues).
- Add or update tests for any new functionality.
- Keep PRs focused — one feature or fix per PR.
- Update the README if your change affects usage or configuration.

## Code of Conduct

Be respectful. We strive to keep this project welcoming for everyone.
